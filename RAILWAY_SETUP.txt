===============================================
RAILWAY DEPLOYMENT SETUP INSTRUCTIONS
===============================================

REQUIRED ENVIRONMENT VARIABLES:
-------------------------------

1. BOT TOKEN (REQUIRED - Use TOKEN or TOKEN_BOT_1):
   - TOKEN=your_discord_bot_token_here
   OR for multiple bots:
   - TOKEN_BOT_1=your_first_bot_token_here
   - TOKEN_BOT_2=your_second_bot_token_here
   
   Get from: Discord Developer Portal > Your Application > Bot > Token
   
   IMPORTANT: The bot only recognizes TOKEN or TOKEN_BOT_* variables.
   Do NOT use other variable names like m1, m2, etc.

2. MONGO (Database):
   MONGO=your_mongodb_connection_string
   
   Get from: MongoDB Atlas > Connect > Connection String
   Example: mongodb+srv://username:password@cluster.mongodb.net/?retryWrites=true&w=majority

3. SPOTIFY CREDENTIALS (Required for Spotify support):
   SPOTIFY_CLIENT_ID=your_spotify_client_id
   SPOTIFY_CLIENT_SECRET=your_spotify_client_secret
   
   Get from: https://developer.spotify.com/dashboard/applications

RECOMMENDED ENVIRONMENT VARIABLES:

DEFAULT_PREFIX=!
AUTO_DOWNLOAD_LAVALINK_SERVERLIST=true
PLAYER_SESSIONS_MONGODB=true

ENABLE_COMMANDS_COOLDOWN=true

DEPLOYMENT STEPS:
-----------------

1. Create a new project on Railway (https://railway.app)

2. Connect your GitHub repository or deploy from CLI

3. Add all required environment variables in Railway:
   - Go to your project > Variables tab
   - Add each variable listed above
   
4. Railway will automatically:
   - Detect Python runtime
   - Install dependencies from requirements.txt
   - Start the bot using the Procfile

5. Monitor deployment:
   - Check the Deployments tab for build status
   - View logs in the Logs tab for any errors

IMPORTANT SECURITY NOTES:
-------------------------

- NEVER commit .env file to GitHub
- Keep your bot token and MongoDB credentials secret
- Regenerate tokens if accidentally exposed
- Use Railway's environment variables feature for all secrets

DISCORD BOT PERMISSIONS REQUIRED:
---------------------------------

Make sure your bot has these intents enabled in Discord Developer Portal:
- Message Content Intent (for reading messages)
- Server Members Intent (if using member features)
- Presence Intent (optional)

And these permissions when inviting:
- Read Messages/View Channels
- Send Messages
- Embed Links
- Attach Files
- Read Message History
- Add Reactions
- Connect (voice)
- Speak (voice)

TROUBLESHOOTING:
----------------

If the bot doesn't start:
1. Check Railway logs for error messages
2. Verify all required environment variables are set
3. Ensure MongoDB connection string is valid
4. Check that bot token hasn't been regenerated

Common errors:
- "Invalid token" - Check TOKEN or TOKEN_BOT_1 variable (make sure it's named correctly!)
- "Cannot connect to MongoDB" - Verify MONGO connection string
- "Missing intents" - Enable required intents in Discord Developer Portal
- Bot crashes immediately - Ensure you used TOKEN or TOKEN_BOT_* (not other names)
